DIY-tek Lens — Modèle de projet

Auteurs : Bérénice CARDOSO-FAUCHET, Pol TYMEN, Lucas REIS-OLIVER, Divine BANON, Alex PEIRANO, Marc DUBOC — Contact : alex.peirano@imt-atlantique.net

Résumé (TL;DR)

DIY-tek Lens est une application mobile qui propose de donner une seconde vie aux objets du quotidien. L’utilisateur prend en photo un objet inutile ou cassé, l’IA reconnaît l’objet et suggère plusieurs idées de projets DIY / low-tech pour le transformer. Objectif : rendre accessible le réemploi créatif et réduire le gaspillage. Format : application open source, interface simple, base d’idées croissante enrichie par la communauté.
Règle utile : une suggestion est utile si elle est réalisable en moins de 2h avec des outils de base.

Contexte et justification

  • Problème constaté : difficulté à trouver rapidement des idées concrètes pour réutiliser des objets du quotidien plutôt que de les jeter.
  • Publics visés : étudiants, familles, makers, fablabs, associations écologiques.
  • Valeur ajoutée : réduire les déchets, proposer des tutoriels simples, favoriser la créativité individuelle et collective.
Exemple concret : une vieille boîte de conserve est transformée en lampe de bureau grâce aux suggestions de l’application.

Objectifs

  • Développer une application mobile avec reconnaissance d’images.
  • Créer une base d’idées DIY/low-tech indexées par objets.
  • Intégrer un mode “communauté” permettant aux utilisateurs de partager leurs projets.
  • Évaluer l’impact écologique : nombre d’objets réemployés vs jetés.
Indicateur clé : au moins 50% des utilisateurs déclarent avoir réalisé un projet concret via l’application.

Fonctionnalités principales (proposition)

Core (Level 0)

  • Prendre une photo et identifier l’objet.
  • Afficher 3 à 5 idées DIY/low-tech simples et illustrées.
  • Accéder à des tutoriels courts.

Extension 1 (Level 1)

  • Mode “communauté” pour partager ses réalisations.
  • Recherche d’idées par type d’objet ou par matériau.

Extension 2 (Level 2)

  • Système de notation et de filtres (temps, difficulté, outils nécessaires).
  • Suggestions locales : matériaux disponibles en ressourceries proches.
Règle utile : chaque idée doit être illustrée d’au moins une photo étape-par-étape.

Documentation et ressources

  • Fiches pas-à-pas (texte + images).
  • Tutoriels vidéo courts (1–3 min).
  • Base collaborative de projets open source.
  • Fiche “sécurité DIY” (outils de base, EPI).
Exemple : tutoriel “Transforme une bouteille plastique en arrosoir” (5 étapes, 15 min).

Parcours utilisateur

  1. Usage express (30 sec) — prendre une photo → 1 idée rapide.
  2. Session standard (15–30 min) — suivre un tutoriel simple → objet transformé.
  3. Projet avancé (1–2h) — idées complexes (mobilier, électronique simple) → livrable final partagé en ligne.
Règle utile : limiter chaque tutoriel à moins de 7 étapes.

Méthodologie

  • Design centré utilisateur (UX simple, 3 clics max pour une idée).
  • IA de reconnaissance d’objets entraînée sur une base ouverte.
  • Contributions communautaires modérées par l’équipe projet.
Règle utile : toujours proposer au moins 1 alternative “sans outils électriques”.

Budget indicatif

  • Développement prototype (3–4 mois) : 10–20 k€.
  • Serveur & API reconnaissance image : 50–150 €/mois.
  • Design graphique et UX : 2–4 k€.
Remarque : coûts réduits en mode open source + contributions bénévoles.

Production et logistique

  • Phase pilote : lancement sur Android (APK) puis iOS.
  • Base de données open source hébergée sur GitHub.
  • Modération communautaire + partenariats avec fablabs et ressourceries.

Tests et validation

  • Beta testeurs : makers, étudiants, familles.
  • Indicateurs : taux de projets réalisés, nombre d’objets sauvés.
  • Feedback : formulaire in-app simple (3 questions).

Gouvernance et licence

  • Repo GitHub public (code + idées DIY).
  • Licence recommandée : MIT pour code, CC-BY-SA pour tutoriels.
  • Rôles clés : développeur IA, designer UX, modérateur communauté, responsable partenariats.

Roadmap 6–12 mois

  1. Prototype 0.1 (photo → 3 idées simples) — M0–M3.
  2. Pilotes utilisateurs (100 bêta-testeurs) — M3–M5.
  3. Ajout mode communauté + recherche par matériau — M6–M8.
  4. Lancement public (Android/iOS) + partenariats — M9–M12.

Risques et contremesures

  • Reconnaissance d’objets imprécise : intégrer option “sélection manuelle”.
  • Manque de contenus : inciter les utilisateurs à contribuer via gamification.
  • Difficulté d’adoption : proposer un mode hors-ligne avec pack d’idées de base.

Mesure d'impact

  • Nombre d’objets transformés (déclarés par les utilisateurs).
  • Poids estimé de déchets évités.
  • Taux d’engagement communautaire (partages, tutoriels créés).
